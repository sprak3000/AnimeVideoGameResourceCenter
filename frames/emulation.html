Emulation

Briefly, an emulator is a program that allows the machine code for a certain microprocessor to be run on a completely different microprocessor. It is a complex task to achieve though; it requires an in-depth knowledge of both the microprocessor you are emulating as well as the microprocessor running the emulator.

What is the point of having an emulator if you don't have any programs to run on it? Well, you could learn the assembly language for the processor being emulated and write your own programs for it. Or you can search around for ROMs or cartridge images of your favorite games from the past. These ROM files are created by a machine that essentially just dumps all the machine code from a cartridge into a computer file. From there, it is up to the emulator to read in that file and execute the code.

If you are looking for emulators or links to ROM sites, look no further than Archaic Ruins or Node 99. They are your one stop shopping center for information on emulators and the emulation scene. The only piece of advice I have is to keep several different emulators for a specific machine on hand. Some emulators run some games better than others.
